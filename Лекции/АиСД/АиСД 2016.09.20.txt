Данные - набор различных информационных объектов, над которыми выполняют те или иные действия операторы программы
Создание структуры данных называется структурированием данных
Выбор структуры данных может в значительной степени повлиять на кач-во программы.
Логическая и физическая структура данных
Логическая СД - представление данных с точки зрения прикладного программиста или пользователей, не зависит от особенностей организации и работы запоминающих устройств
Физическая СД - реальные способы хранения данных в ЭВМ

На логическом уровне описания данных различают следующее:
	Поле - наименьшая именованная единица данных, может состоять из любого кол-ва битов/байтов, которые имеют идентификатор
		Обычно поле содержит значение характеристики объекта предметной области
	Аттрибут - характеристика объекта в предметной области (Реквизит)
		Характеризуется именем, типом, значением и диапазоном(ОДЗ)
	Домен - диапазон аттрибута
	Запись(логическая) - именованная совокупность связанных полей, рассматриваемых как одно целое
		Обычно содержит описание объекта предметной области
		Аттрибуты, используемые для идентификации записей называют ключами, может состоять из одного или нескольких аттрибутов, значение ключа есть значение соответствующего аттрибута(аттрибутов)
		Запись должна однозначно определяться значением ключа
		Длина записи равно количеству исп памяти и равна кол-ву памяти
		Различают записи фиксированной, переменной и неопределённо длины
		Записи фикс длины имеют заранее известную длину
		Записи перем длины указывают длину в самой записи
		Записи неопр длины, где длина записи явно не указывается, а в конце записи стоит символ (маркер)
	Массив - набор однородных элементов (простых элементов данных или записей), с каждым из которых связан упорядоченный набор целых чисел, называемых индексами
		Индексы однозначно определяют позицию каждого элемента массива
		Одномерные и многомерные
		n-мерный
	Файл (логический) - именованная совокупность записей, рассматриваемых как единое целое
На физическом:
	бит - наим един инф
	байт
	слово - совокупность разрядов, образующих наименьшую единицу информации, передаваемую между памятью и процессором
	физическая запись - единица данных на физическом носителе, которая может быть считана или записана одной командой ввода-вывода эвм (те данные, которые записываются между двумя межблочными промежутками на магнитной ленте или адресными маркерами на диске)
		одна физ - неск логич
	блок - группа данных, составляющих физическую запись, называются блоком
	блок - совокупность премыкающих друг к другу хранимых логических записей, записанных как единое целое
	коэф блокирования - кол-во записей, нах. в блоке
	экстент - совокупность физ записей, расположенных последовательно на внешнем запоминающем устройстве. Логическая взаимосвязь заисей в экстенте не обязательна
	набоор данных
	файл - совокупность физ записей (блоков), обрабатываемых в эвм под одним именем. набор данных может занимать один или несколько экстендов
	файл может содержаться внутри одного тома или размещаться на нескольких томах
	том - отдельная физ единица любой внешней запоминающей среды, напр. катушка магнитной ленты, пакет дисков, магнитный барабан
	распределение данных - операция распределения участков памяти для хранения данных
		статическое и динамическое
		статическое - где занимают фикс зоны на время всего решения задачи
		если сд в ходе решения задачи измеяют свою длину и конфигурацию, то распределение и перераспределение памяти происходит и по ходу выполнения программы и наз динамическим
	Если рассматривать СД с точки зрения их локализации в выч машине:
		в основной(оперативной) памяти - оперативные структуры
		структуры данных во внешней памяти
