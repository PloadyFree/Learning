Классификация АСК по составу и сложности команд
	Архитектура с полным набором команд (CISC - complex instuctions set of computer)
	Архитектура с сокращённым набором команд (RISC - reduced instruction set computer)
	Архитектура с командными словами сверхбольшой длины (VLIW - very long instruction word)
	
	1. CISC:
		IBM 360
		i8086...
		Особенности:
			1. Малое количество регистров общего назначения (РОН)
			2. Большое количество команд
			3. Разнообразие способов адресации
			4. Множество форматов
			5. Наличие команд, где обработка совмещена с доступом к памяти
		Статистика:
			Количество сложных команд не превышает 10-20%
			Количество наиболее сложных команд не превышает 0.2%
			В сумме занимают до 60% места
	2. RISC
		Особенности:
			1. Команд мало, команды наиболее часто используемы
			2. Большое количество регистров общего назначения (РОН)
			3. Отстутствие команд, совмещающих обработку и доступ к памяти
			4. Фиксированный формат команд
			5. Простота дешефрирования команд
		CDC-6600 CRAY (суперкомпьютер)
		Alpha
	3. VLIW
		Архитектура RISC с объединением нескольких RISC-команд в одну сверхбольш
	
Классификация АСК по месту хранения операндов
	Стековая архитектура
		a = a + b + ac
		push a
		push b
		add
		push a
		push c
		mul
		add
		pop a
		
		Нуль- и один- операнды
		Сокращение адресной части
	
	Аккумуляторная архитектура
		Добавляется регистр аккумулятор
		
		d = c / (a + b)
		
		load a 	=> a в аккумуляторе
		add b 	=> a+b в аккумуляторе
		store d	=> в d лежит a+b
		load c	=> c в аккумуляторе
		div d	=> c/d в аккумуляторе
		store d	=> в d лежит c / (a + b)
		
		Простота декодирования
		Дешёвая архитектура
		
		Многократное обращение к памяти
	
	Регистровая архитектура
		Регистровый файл:
			Регистр общего назначения (РОН)
		Разнообразие способов адресации:
			регистр-регистр
			регистр-память
			п-р
			р-непосредственный операнд
			п-н
			п-п
	
	Архитектура с выделенным доступом к памяти
		Характерна для RISC-процессоров
		load р, п
		store п, р
		
		add R1, R2 => результат в R1
	
Типы и форматы операндов
	Базовые типы операндов:
		Числа
		Символы
		Логические данные
		Адреса
	Числа с фиксированной запятой
		знак|a[n-1]|...|a[1]|a[0]||a[-1]|a[-2]|...|a[-n]
		n=0
		k=0 - целые числа
		
		a[n-1]|a[n-2]|...|a[1]|a[0]
		знак|a[n-2]|...|a[1]|a[0]
		Формат			Размер		Знаковый вариант
		байт			8 бит		+
		слово			16			+
		двойное слово	32			+
		бл				32			-
		дал				48			-
