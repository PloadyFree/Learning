(angtos <d> [ <Flag> [<length>]]) - угол в строку
	Представление числа через мантиссу и порядок называется научным
	Форматы преобразования (флаг)
		0 - градусы
		1 - градусы / минуты / секунды
		2 - грады
		3 - радианы
		4 - геодезические единицы
	Точность
		Для всех форматов кроме 3 определяет количество десятичных разрядов дробной части
		Для формата 3 - общую длину числа
(ASCII <char>) - получить код символа
(CHR <Int>) - получить символ по коду
(atof <str>) - строку в действительной
(atol <str>) - строка в целое
(div <Real>) - целая часть действительного числа
(float <Int>) - преобразует целое в действительное
(itoa <Int>) - целое в строку
(rtos <Real> [<Flag> [<length>]]) - действительное в строку
	Flag
		1 - вид числа в строке - научный
			Длина определяет длину мантисы
		2 - десятичный
			Длина определяет количество десяти разрядов после точки
			Пример:
				13.867, точность 2 => 13.87
			При выводе происходит округление числа
		3 - инженерные футы и десятичные дюймы, при этом 2 и 3 не отличаются друг от друга
		4 - архитектурный (художественный) - футы и дробные дюймы
			2.75 4 1 => 2 3/4
		5 - произвольная дробная часть - примерно аналогично режиму 4

Работа с файлами
	Файлом называются данные, хранящиеся на некотором периферийном устройстве
	Файл характеризуется двумя параметрами:
		Типом информации (обработки) в файле
		Размером буфера
	Буфером называется определённая часть оперативной памяти, через которую происходит обмен информации между перефирийным устройством и центральным процессором
	В современных компьютерах для этого выдеояется специальная быстрая память - кэш
	Каждый файл обладает своим уникальным буфером
	Размер буфера определяется ОС
	Для ОС фирмы Microsoft размер буфера равен 512 байтам, в UNIX-подобных системах размер буфера задаётся системным программистом в начале сеанса
